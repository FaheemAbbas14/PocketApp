# POCKET APP - BRAND GUIDELINES

## Color Palette

### Primary Colors
- **Primary Blue**: #1A76A1 (RGB: 26, 118, 161)
  - Used for main UI elements, headers, buttons
  - Professional and trustworthy appearance
  
- **Secondary Green**: #4CAF50 (RGB: 76, 175, 80)
  - Used for success states, completed items
  - Represents completion and achievement

- **Accent Orange**: #FF9800 (RGB: 255, 152, 0)
  - Used for alerts, important information
  - Draws attention to key actions

### Neutral Colors
- **White**: #FFFFFF (RGB: 255, 255, 255)
  - Background, text on dark elements
  
- **Light Gray**: #F5F5F5 (RGB: 245, 245, 245)
  - Secondary backgrounds
  
- **Dark Gray**: #333333 (RGB: 51, 51, 51)
  - Primary text color

## Typography

### Font Family
- **Android**: Roboto (system default)
- **Custom**: Consider using:
  - Montserrat for headers
  - Open Sans for body text

### Font Sizes
- **App Title**: 24-32 sp
- **Section Headers**: 18-20 sp
- **Body Text**: 14-16 sp
- **Small Text**: 12 sp

## Logo & Icon Guidelines

### Logo Style
- Simplified pocket icon with "P" lettermark
- Maintains legibility at all sizes
- Works in both light and dark modes

### Adaptive Icon
- Foreground: Pocket shape with letter "P"
- Background: Solid primary blue color
- Follows Google Material Design 3 specifications

## Module Icons and Color Associations

### Task Management
- Color: Primary Blue (#1A76A1)
- Icon: Checkmark/Checklist
- Represents: Planning and completion

### Expense Tracking
- Color: Secondary Green (#4CAF50)
- Icon: Dollar sign/Wallet
- Represents: Money and financial control

### Event Planning
- Color: Accent Orange (#FF9800)
- Icon: Calendar
- Represents: Important dates and timing

### Payment Management
- Color: Primary Blue variant
- Icon: Payment card/Bank
- Represents: Financial transactions

## UI Patterns

### Buttons
- Primary Button: Primary Blue background, white text
- Secondary Button: Light Gray background, dark text
- Danger Button: Red background, white text

### Cards
- White background with subtle shadow
- 8dp border radius
- Primary color accent on left edge

### Screens
- Light background (white or light gray)
- Dark text for readability
- Consistent padding: 16dp

## Asset Dimensions

### Icons
- 24x24 dp - Navigation, small icons
- 48x48 dp - Medium icons
- 72x72 dp - Large feature icons
- 108x108, 192x192, 512x512 - App icons

### Screen Dimensions
- Phone: 1080x1920 (9:16)
- Tablet: Responsive to screen size

## Accessibility

- Contrast ratio minimum 4.5:1 for text
- Icon sizes minimum 48x48 dp for touch targets
- Color is never the only indicator
- All images have descriptive alt text

## Marketing & Promotional Use

### Social Media
- Use bright, engaging colors
- Include app icon and main features
- Maintain consistent branding

### App Store Listing
- Professional, clear imagery
- Showcase key features
- Maintain visual hierarchy

## Implementation Notes

All generated assets follow these guidelines. When creating custom assets:
1. Maintain color consistency
2. Use proper icon sizes
3. Follow Material Design 3 principles
4. Test on multiple devices
5. Ensure accessibility compliance
